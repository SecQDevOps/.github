# Security Policy

## Reporting a Vulnerability

We take the security of QNODE and the SecQDevOps platform seriously. If you discover a security vulnerability, please report it **privately** to:

📧 **[secqdevops@hs-rm.de](mailto:secqdevops@hs-rm.de)**

### Please Do Not

- **Do not** open a public GitHub issue for security vulnerabilities.
- **Do not** disclose the vulnerability publicly until we've had time to assess and patch it.

### What to Include

When reporting a vulnerability, please provide:

1. **Description:** A clear, detailed explanation of the vulnerability.
2. **Affected component:** Which repository, service, or module is affected.
3. **Steps to reproduce:** Minimal steps or proof-of-concept code to demonstrate the issue.
4. **Potential impact:** What could be exploited and what are the consequences?
5. **Your contact information:** How we can reach you (email, GitHub handle, etc.).

## Vulnerability Response Process

1. **Acknowledgment:** We will acknowledge receipt within 5 business days.
2. **Assessment:** Our team will investigate and determine the severity and scope.
3. **Patch development:** We will work to develop and test a fix.
4. **Responsible disclosure:** We will coordinate with you on a timeline for public disclosure.
5. **Release:** Security patches will be released with clear communication to users.

## Supported Versions

Security updates are provided for the latest released version of QNODE. The project is actively developed, and we recommend users stay up to date.

## About This Project

**QNODE** is part of the **SecQDevOps** project, funded under the EU Horizon Europe Research and Innovation Programme (Grant Agreement No. 101225776). Our commitment to security extends across all layers of the stack, from cryptography and circuit validation to edge node orchestration.

## Questions?

For security-related questions or clarifications, contact [secqdevops@hs-rm.de](mailto:secqdevops@hs-rm.de).

Thank you for helping us keep QNODE and SecQDevOps secure!
